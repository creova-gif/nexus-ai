# NexusAI Technical Architecture

## Overview

NexusAI is an enterprise-grade web platform designed to serve Canadian banking stakeholders with AI-powered compliance, fraud detection, personalized financial advisory, and open banking capabilities. The platform is built on a modern, scalable architecture that prioritizes security, performance, and user experience.

## Technology Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Frontend** | React 19 + TypeScript | Modern, reactive UI framework |
| **Styling** | Tailwind CSS 4 + shadcn/ui | Elegant, responsive design system |
| **Backend** | Express 4 + Node.js | Lightweight, scalable server |
| **API** | tRPC 11 | Type-safe, end-to-end API framework |
| **Database** | MySQL/TiDB | Relational database for structured data |
| **ORM** | Drizzle ORM | Type-safe database queries |
| **Authentication** | Manus OAuth + JWT | Secure user authentication |
| **AI/LLM** | Manus Built-in LLM API | Compliance insights, financial advisory, sentiment analysis |
| **Storage** | S3 (via Manus) | File storage for documents and reports |
| **Real-time** | Socket.io (optional) | Real-time alerts and notifications |

## Database Schema

The database is organized into eight core entity groups:

### 1. Authentication & Users
- **users**: Core user table with OAuth integration, roles, and profile data
- **sessions**: Session management for authentication state

### 2. Compliance & AML
- **transactions**: Financial transaction records with metadata
- **aml_alerts**: AML alerts with risk scores and investigation status
- **alert_investigations**: Investigation records linked to alerts
- **suspicious_activity_reports**: SAR documents generated by compliance officers

### 3. Financial Advisory
- **customer_profiles**: Customer financial profiles and preferences
- **spending_insights**: AI-generated spending analysis and patterns
- **financial_goals**: Customer-defined financial goals
- **investment_recommendations**: AI-driven investment suggestions

### 4. Open Banking
- **connected_institutions**: Third-party financial institutions
- **data_sharing_consents**: User consent records for data sharing
- **aggregated_accounts**: Aggregated account data from connected institutions

### 5. Customer Support
- **support_conversations**: Chat history with AI assistant
- **support_tickets**: Escalated support cases
- **sentiment_analysis**: Sentiment scores for customer interactions

### 6. Admin & Monitoring
- **audit_logs**: System audit trail
- **platform_analytics**: Usage metrics and KPIs
- **system_health**: System performance monitoring

## User Roles & Permissions

| Role | Access Level | Primary Responsibilities |
|------|--------------|--------------------------|
| **Bank Admin** | Full platform access | User management, system configuration, analytics |
| **Compliance Officer** | Compliance & AML workspace | Alert investigation, SAR generation, reporting |
| **Financial Advisor** | Advisory workspace | Customer insights, recommendations, goal tracking |
| **Retail Customer** | Personal workspace | Financial overview, advisory, support, open banking |

### Design System

The platform uses a premium, refined design language:

- **Color Palette**: Deep blues (trust, stability) + vibrant greens (growth, security) + accent grays
- **Typography**: Clean, modern sans-serif fonts (geometric for headings, humanist for body)
- **Spacing**: 8px base unit with consistent rhythm
- **Shadows**: Soft, layered shadows for depth
- **Interactions**: Smooth transitions, clear feedback, accessible focus states

## Security Considerations

1. **Authentication**: Manus OAuth with JWT session cookies
2. **Authorization**: Role-based access control (RBAC) enforced at API level
3. **Data Protection**: All sensitive data encrypted in transit (HTTPS) and at rest
4. **Audit Logging**: All compliance-related actions logged for regulatory compliance
5. **API Security**: tRPC procedures validate input and enforce permissions
6. **CORS**: Configured for secure cross-origin requests

## AI Integration Points

1. **AML Alert Generation**: LLM analyzes transaction patterns for anomalies
2. **Spending Insights**: LLM generates personalized financial analysis
3. **Investment Recommendations**: LLM provides tailored investment suggestions
4. **Sentiment Analysis**: LLM analyzes customer support conversations
5. **SAR Generation**: LLM assists in drafting suspicious activity reports

## Deployment & Hosting

- **Frontend**: Deployed via Manus (automatic builds from repository)
- **Backend**: Node.js server running on Manus infrastructure
- **Database**: MySQL/TiDB managed by Manus
- **Storage**: S3 via Manus for file storage
- **Domain**: Custom domain support via Manus

## Development Workflow

1. **Schema First**: Define database tables in `drizzle/schema.ts`
2. **Migration**: Generate SQL via `pnpm drizzle-kit generate`
3. **Query Helpers**: Add database queries in `server/db.ts`
4. **API Procedures**: Define tRPC procedures in `server/routers.ts`
5. **Frontend UI**: Build React components consuming tRPC hooks
6. **Testing**: Write Vitest specs for critical paths
7. **Checkpoint**: Save progress via `webdev_save_checkpoint`

## Performance Optimization

- **Code Splitting**: Lazy-load role-specific pages
- **Query Optimization**: Indexed database queries for fast lookups
- **Caching**: tRPC query caching with React Query
- **Image Optimization**: Use S3 for media with proper formats
- **Bundle Size**: Tree-shaking and minification via Vite

## Monitoring & Analytics

- **Platform Analytics**: Track user engagement, feature adoption
- **System Health**: Monitor server performance, database health
- **Audit Logs**: Compliance-focused logging for regulatory requirements
- **Error Tracking**: Capture and report runtime errors
